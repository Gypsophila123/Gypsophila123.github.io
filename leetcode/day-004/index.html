题目

283. 移动零

难度简单 859收藏

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

示例:

说明:

  1. 必须在原数组上操作，不能拷贝额外的数组。
  2. 尽量减少操作次数。

我的思路

今天的题目非常简单，起码看起来是这样。简单的要求，简短的题目，但往往越是简单的题目越是能体现出差距。我已经准备好被那些大神们血虐了。

原本我还以为非零元素需要排序，重审之后发现仅仅需要移动零即可，难点在于不能拷贝数组且非零元素要保持原顺序。

第一个思路来的很快，从后往前遍历数组，遇到零就将其一位一位的向后移。这样就可以保证顺序且没有拷贝数组，当然，我感觉这样会比较费时，但我还没想出更好的解决方案，先写写吧！

写的很顺利，没出什么岔子，一次过。但是结果嘛

只能说都在预料之中，现在有两条路，在此基础上尝试优化，或者直接换个思路。

不过我盲猜大佬们用时都在 5ms 以内，看来直接早点换个思路才是正解。

刚才直接使用暴力方法，一个一个的去换位置，这样虽然简洁，但是效率却高不起来，那么有没有什么更加巧妙的方法呢？

经过一番观察与思考，我想出一个更简单的方法，而且感觉思路更简单，如下：

输入： [0,1,0,3,12]

  1. [1,1,0,3,12]
  2. [1,3,0,3,12]
  3. [1,3,12,3,12]
  4. [1,3,12,0,0]

简单点说就是找到非 0 数字，直接一个个的放进去，找到末尾就在后面补零。参考上面应该很容易理解，这样免去了很多麻烦，而且遍历一次就保证完成。不多说了，开写

虽然早有预料，但是看到这个击败100%用户的时候我的心里还是非常高兴的，我是不是也能在简单场叱咤风云了（笑）。这回我不是猪脑袋了

大神解答

王尼玛发布于 2020-02-0941.3k精选PythonJava

两次遍历

我们创建两个指针i和j，第一次遍历的时候指针j用来记录当前有多少非0元素。即遍历的时候每遇到一个非0元素就将其往数组左边挪，第一次遍历完后，j指针的下标就指向了最后一个非0元素下标。
第二次遍历的时候，起始位置就从j开始到结束，将剩下的这段区域内的元素全部置为0。

一次遍历

这里参考了快速排序的思想，快速排序首先要确定一个待分割的元素做中间点x，然后把所有小于等于x的元素放到x的左边，大于x的元素放到其右边。
这里我们可以用0当做这个中间点，把不等于0(注意题目没说不能有负数)的放到中间点的左边，等于0的放到其右边。
这的中间点就是0本身，所以实现起来比快速排序简单很多，我们使用两个指针i和j，只要nums[i]!=0，我们就交换nums[i]和nums[j]
