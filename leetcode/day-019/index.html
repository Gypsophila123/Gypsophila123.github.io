题目

将数组拆分成斐波那契序列

难度中等

给定一个数字字符串 S，比如 S = "123456579"，我们可以将它分成斐波那契式的序列 [123, 456, 579]。

形式上，斐波那契式序列是一个非负整数列表 F，且满足：

  • 0 <= F[i] <= 2^31 - 1，（也就是说，每个整数都符合 32 位有符号整数类型）；
  • F.length >= 3；
  • 对于所有的0 <= i < F.length - 2，都有 F[i] + F[i+1] = F[i+2] 成立。

另外，请注意，将字符串拆分成小块时，每个块的数字一定不要以零开头，除非这个块是数字 0 本身。

返回从 S 拆分出来的任意一组斐波那契式的序列块，如果不能拆分则返回 []。

我的思路

首先来研究一下斐波那契数列，根据题目可以得知必须满足：

对于所有的0 <= i < F.length - 2，都有 F[i] + F[i+1] = F[i+2] 成立。

才被称之为斐波那契数列，所以这个数列最少也会有三位数，那么我们现在有几个问题：

  • 怎么分割字符串才能保证可行

可以看到数字的长度不是固定的，我们唯一知道的就是 F[i] + F[i+1] = F[i+2]

我们只要找到开头三个数，实际上确定了开头后面的都可以通过计算得出。

  • 怎么才能穷举所有情况

根据示例，可以看到情况非常复杂，我们怎么样才能将所有可能都穷举一遍呢？

我的思路是由短到长，最短长度是 1，而最大长度：
只要第一和第二两数中最长的一个长度不超过剩余长度（第三个数）即可（因为数字开头不能为 0）。

同时因为“每个整数都符合 32 位有符号整数类型”，所以如果长度过长（某个数超过 2的 31次方 - 1，我们就直接跳过）

如果上面这两个问题能解决，那么代码结构就是：先确定头三个数 -> 根据头三个数演算直到结束。
如果有一步对不上就重新确定头三个数，一直到穷举完所有可能。

这样的话代码很不容易写，而且有个问题，如何穷举？因为我们不知道这三个数长度的规律，穷举次数可能会达到恐怖的程度，所以我们研究一下其长度

第一个数和第二个数的长度是完全未知的。只要在规定内都有可能。
第三个数长度可能等于前两个数中较长的那个，也可能等于较长那个加 1

准确范围：

从例 5 看出，第一个数长度最大为字符串长度的一半 减 1，此时第二个数长度为 1。反之亦然

本质是： 数 3 的 长度或 数 1 和数 2 中较长数或者较长数加 1
所以数 3 会占总长度的 1 / 3 ~ 1 / 2

代码一直缝缝补补的，然后就变成这副鬼样子了。。。可以说是非常之烂了，而且成绩更烂。
