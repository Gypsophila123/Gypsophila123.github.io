题目

写一个函数，求两个整数之和，要求在函数体内不得使用 “+”、“-”、“*”、“/” 四则运算符号。

我的思路

又是简短的题目，题目非常简单，不用四则运算搞定两数求和，这两个数可以是任何数。看到第一眼，感觉这个题就是在考查计算机是怎么实现加法的。

在脑海里搜索这个问题的答案，补码？不对不对，这是用来存数的。与门？想了想这也不对啊！先找个两个二进制数模拟下：

  • 原数

0100
0110

  • 先是两个零，不动

0010
0110

0000

  • 一个一，直接拿过去

0100
0100

0010

  • 两个一，进一位，原位置归零

0000
0000

1010

轻松得出一点小结论：两个数相同归零，不同为一；对的，就是异或。

不过不要大意，因为我们还没算进位，看来这个题比我想象的要难。可以看到上面的例子只有一次进位，即两个 1 时，

方法呼之欲出，就是 与 运算，但是我们一个异或已经算好了所有位了呀！怎么才能在一个已经算好的数上用与运算进位呢？我苦思冥想了许久，这正是这道题的难点所在。老办法，还是列出来，光靠想是想不出来的。

  • 依旧是原来那两个数

  • 进行异或运算后

0010

  • 进行与运算后

0100

  • 而我们要得到的是

1010

观察一番后我可以轻松的说，与运算后左移再加异或运算，emmmm，好像不能用加法来着？啊这怎么办？

依稀记得以前学电路的时候学过全加器，难道要用噩梦一般的电路来解了？

到这之后我真没想出来，偷偷去看了眼解答

其实我陷入了一个思维误区，虽然规定不能用加，但是我们实现的就是一个加法运算，所以说我们可以用递归来解决这个问题。那什么时候退出递归呢？很简单，我们只有在需要进位的时候才要用到这个加，那不用进位就可以返回了，简直是一个天才的想法。

那么开始写代码吧！

面试题65. 不用加减乘除做加法（位运算，清晰图解）

解题思路：

本题考察对位运算的灵活使用，即使用位运算实现加法。
设两数字的二进制形式 a, ba,b ，其求和 s = a + bs=a+b ，a(i)a(i) 代表 aa 的二进制第 ii 位，则分为以下四种情况：

a(i)a(i) b(i)b(i) 无进位和 n(i)n(i) 进位 c(i+1)c(i+1)
00 00 00 00
00 11 11 00
11 00 11 00
11 11 00 11

观察发现，无进位和 与 异或运算 规律相同，进位 和 与运算 规律相同（并需左移一位）。因此，无进位和 nn 与进位 cc 的计算公式如下；

\begin{cases} n = a \oplus b & 非进位和：异或运算 \ c = a & b << 1 & 进位：与运算 + 左移一位 \end{cases}{n=a⊕b**c=a&b<<1非进位和：异或运算进位：与运算+左移一位

（和 ss ）==（非进位和 nn ）++（进位 cc ）。即可将 s = a + bs=a+b 转化为：

s = a + b \Rightarrow s = n + cs=a+b⇒s=n+c

循环求 nn 和 cc ，直至进位 c = 0c=0 ；此时 s = ns=n ，返回 nn 即可。

Q ： 若数字 aa 和 bb 中有负数，则变成了减法，如何处理？
A ： 在计算机系统中，数值一律用 补码 来表示和存储。补码的优势： 加法、减法可以统一处理（CPU只有加法器）。因此，以上方法 同时适用于正数和负数的加法 。
