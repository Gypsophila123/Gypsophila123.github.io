题目

有两种特殊字符。第一种字符可以用一比特0来表示。第二种字符可以用两比特(10 或 11)来表示。

现给一个由若干比特组成的字符串。问最后一个字符是否必定为一个一比特字符。给定的字符串总是由0结束。

我的思路

看了第一眼，我还以为这道题是判断数组长度。当然我不可能这么粗心，仔细审题后发现：第一种字符可以用一比特0来表示。第二种字符可以用两比特(10 或 11)来表示。原来只用了一半，那没事了。

情况还是很简单，只需看最后三位数嘛！由0结尾，即共有四种情况：010、100、110、000。由规定可知 010 结尾必为2bit字符，而 100、000 结尾必为1比特字符。

问题出在 110 （开门，查水表）。 如果结尾是110 就有两种情况了，要判断这个问题只能向钱（前）看，假如倒数第四位是 0 则结尾为1bit字符，是 1 就再向前看，假如倒数第五位是 0 则结尾为2bit字符，是 1 就再向前看…

我称这个环节为找零，在双数为1bit，单数则为2bit。思路清晰了就开始写代码吧！

写到这里，我发觉到了不对劲，前置的判断条件太多了else里面还要嵌套，写算法题不该这样的。其实我本身就不喜欢写if嵌套，这会让我的代码看起来不那么简洁，太多的逻辑判断也容易疏漏，我试图寻找更简单的结构。

由于结尾必为 0 所以不予考虑，如果有倒数第二位且为 0 则结尾必为 1bit，为 1 则继续向前找零，一直循环下去…

好了，简化思路继续前行

写完之后自己试了下感觉没问题就提交了，然后直接过了（有点不相信），以前写题一次过的情况不多，Nice！可以早点睡了。
